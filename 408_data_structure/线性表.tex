\section{线性表的定义}
\begin{cquote}
  线性表是n个类型相同的数据元素的有限序列，对$n>0$，除第一元素无直接前驱、最后一个元素无直接后继外，
  其余的每个数据元素只有一个直接前驱和一个直接后继。线性表是最简单、最常用的数据结构之一，是其他数据结构的基础。
\end{cquote}

使用更专业一点的语言表达：线性表具有{\color{ctp-rosewater}{相同数据类型}}的n(n>=0)个数据元素的{\color{ctp-rosewater}{有限序列}}，
其中n为{\color{ctp-red}{表长}}，当n=0时线性表是一个空表。
若用L命名线性表，则一般表示为：$L = (a_1, a_2, \dots, a_n)$。

线性表中的数据元素{\color{ctp-blue}{类型是相同的}}，如果用C++中类型来说，那就是最常用的三个容器：\code{std::vector}、\code{std::list}和\code{std::forward\_list}。
线性表中的数据元素是{\color{ctp-blue}{有序的}}，在生活中有非常多地方可以看到使用线性表存储和表达的场合。比如：学生名单、图书目录、购物清单、待办事项列表等等。
线性表中的数据元素是{\color{ctp-blue}{有限的}}，线性表的长度是有限的。线性表中的数据元素是有序的，线性表中的每个数据元素都有一个确定的位置。
线性表中的数据元素是{\color{ctp-blue}{相同类型的}}，线性表中的数据元素类型可以是基本数据类型，也可以是用户自定义的数据类型。

线性表中的{\color{ctp-red}{$a_i$}}表示线性表中第\code{i}个元素，可以称作“位序”。
在使用C++的编写的时候，可以写成\code{arr[i-1]}，因为C++中的数组是从0开始的。也可以写成\code{arr.at(i-1)}，这样可以增加越界检查。

\section{线性表的基本操作}
\code{InitList(L)}：初始化表，构造一个空的线性表L，分配内存空间。
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：L为未初始化线性表。
  \item[] \hspace{1.5em}操作结构L：将L初始化为空表。
\end{itemize}

\code{ListLength(L)}：获取线性表L的长度。
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：L为已初始化的线性表。
  \item[] \hspace{1.5em}操作结果：返回线性表L的长度。
\end{itemize}

\code{GetData(L, i)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：表L已经存在，前 i<= i <= ListLength(L)
  \item[] \hspace{1.5em}操作结果：返回线性表L中第i个合法元素的值。
\end{itemize}

\code{InsList(L, i, e)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：表L已经存在，e为合法元素值且1<=i<=ListLength(L) + 1。
  \item[] \hspace{1.5em}操作结果：在L中第i个位置之前插入新的数据元素e，L的长度加1。
\end{itemize}

\code{DelList(L, i, e)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：表L已经存在且非空，1<=i<=ListLength(L)。
  \item[] \hspace{1.5em}操作结果：删除L的第i个数据元素，并用e返回其值，L的长度减1。
\end{itemize}

\code{Locate(L, e)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：表L已经存在且非空，1<=i<=ListLength(L)。
  \item[] \hspace{1.5em}操作结果：删除L的第i个数据元素，并用e返回其值，L的长度减1。
\end{itemize}

\code{DestroyList(L)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：线性表L已经存在。
  \item[] \hspace{1.5em}操作结果：将L销毁。
\end{itemize}

\code{ClearList(L)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：线性表L已经存在。
  \item[] \hspace{1.5em}操作结果：将L置为空表。
\end{itemize}

\code{EmptyList(L)}
\begin{itemize}
  \item[] \hspace{1.5em}操作前提：线性表L已经存在。
  \item[] \hspace{1.5em}操作结果：如果L为空表则返回\code{TRUE}，否则返回\code{FALSE}。
\end{itemize}

\subsection{简单总结}
线性表分为顺序存储和链式存储两大类，顺序存储只有顺序表这一中结构。
而链式存储可以分为单链表、双链表、循环链表和静态链表四个数据结构，前三种依靠指针实现，
而静态链表借助数组实现。

线性表是算法题命题的重点，此类算法题的实现比较容易且代码量较少。
按照考试的情况，需要在算法时间复杂度或算法空间复杂度上由有效的表现才能够拿满分，
这就比较困难，因此在时间紧迫的情况下， 直接使用暴力法，所以还要多刷刷力扣。

线性表是一种逻辑结构，表示元素之间一对一的相邻关系，顺序表和链表是指存储结构，
两者属于不同的概念、因此不要将其混淆。

线性表的基本操作的实现取决于采用哪种存储结构，存储结构不同，算法也不同。
比如对于线性表进行插入元素操作\code{InsertElem(L)}，参数L，可以使用指针，也可以使用引用。

\section{线性表的顺序表示}
\subsection{顺序表的定义}
线性表的顺序存储也称顺序表，它是用一组地址连续的存储单元以此存储线性表中的数据元素，
从而使得逻辑上相邻两个元素在物理位置上也相邻。因此也可以说，顺序表的特点是表中元素的
逻辑顺序与其存储的物理顺序相同。

举个例子，有一个C++数组\code{std::array<int, 120> someData;}，定义了一个变量someData，
在里面分配了120个int类型的元素。在代码层面上，我们可以使用\code{someData.at(0)}，
\code{someData.at(1)} ... \code{someData.at(119)}。someData中的数据元素可以使用下标获取位置上的值，
这在逻辑上是连续的， 而且内存上也是按照顺序进行排列，加入一个\code{int}占用8个字节，
那么想要获得第一个元素那么在内存地址中从0位可以访问，第四个元素的位置就是\code{0+index * sizeof(int)}。

\begin{itemize}
  \item index = 0, addr = 0
  \item index = 1, addr = 0 + sizeof(int)
  \item index = 2, addr = 0 + 2 * sizeof(int)
  \item index = maxSize - 1, addr = 0 + (maxSize - 1) * sizeof(int)
\end{itemize}

因此进行总结，获取数组下标为n的元素，其内存地址为：LOC(A) + n * sizeof(ElemType) \\
每个数据元素的存储位置都和存储表的起始位置相差一个和该元素的位序成正常比的常数，
因此，顺序表中的任意一个数据元素都可以随机存取，所以线性表的顺序存储结构是一种随机存储的存储结构。
通常用高级程序设计语言中的数据来描述线性表的顺序存储结构。 \\
需要注意的是元素的位序是从1开始的，而数组中的元素的下标是从0开始的。

手动实现一个静态分配的顺序表结构描述
\begin{lstlisting}[language=C++]
#define MaxSize = 50
typedef struct{
  ElementType data[MaxSize];
  int length;
}SqList;

template<Typename T, int size>
using SqList = struct{
  T data[size];
  int length;
}
\end{lstlisting}

\subsubsection{顺序表的初始化}

\begin{lstlisting}[language=C++]
  void init_list(sq_list &L) {
    L.length = 0;
  }
\end{lstlisting}

静态分配和动态分配的顺序表的初始化操作是不同的。静态分配在声明一个顺序表，就已经为其分配了内存空间。
因此初始化时只需要将长度置为0即可。而动态分配的顺序表在初始化时需要为其分配内存空间，通常是通过malloc函数来实现的。
在C++中两个容器\code{std::vector}和\code{std::array}的初始化也是不同的，
前者需要调用构造函数来分配内存空间，而后者在声明时就已经分配了内存空间。
由此可见，静态分配的顺序表在初始化时更简单一些。
\begin {lstlisting}[language=C++]
void init_list(sq_list *L) {
  L->data = (ElementType *)malloc(MaxSize * sizeof(ElementType));
  L->length = 0;
  L->max_size = MaxSize;
}
\end{lstlisting}

\subsubsection{插入操作}
在顺序表的L的第i的位置插入新元素。这需要考虑到顺序表结构的性质。
首先需要确定插入的位置是不是小于0或者大于表长度，那么插入就不合法了。
其次需要考虑到顺序表的长度是不是已经达到最大值，此时再插入就会导致内存溢出。
此外，在进行插入时，需要将插入位置之后的元素都向后移动一位，以腾出插入位置的空间。
\begin{lstlisting}[language=C++]
bool insert_list(sq_list &L, int i, ElementType e) {
  if (i < 1 || i > L.length + 1) {
    return false; // 插入位置不合法
  }
  if (L.length >= MaxSize) {
    return false; // 顺序表已满，无法插入
  }
  for (int j = L.length; j >= i; j--) {
    L.data[j] = L.data[j - 1]; // 将插入位置之后的元素向后移动一位
  }
  L.data[i - 1] = e; // 插入新元素
  L.length++; // 更新顺序表长度
  return true; // 插入成功
}
\end{lstlisting}
最好的插入情况是直接在表尾插入数据，这样不需要向后移动元素，时间复杂度为O(1)。 \\
最坏的插入情况是在表头插入数据，这样需要将所有元素都向后移动一位，时间复杂度为O(n)。平均情况下，插入操作的时间复杂度为Ow(n)。 \\
因此平均时间复杂度就是假设一个元素从第0位开始插入的复杂度加上第1位插入的复杂度加上第2位插入的复杂度...加上第n-1位插入的复杂度，最后除以n。
最后得到插入算法的平均时间复杂度为O(n)。

\subsubsection{删除操作}
删除操作和插入操作类似，需要考虑删除位置是否合法，删除后需要将删除位置之后的元素都向前移动一位，以填补删除位置的空缺。
具体来说要判断index是否<0或者>表长，如果是的话删除位置不合法了。其次需要考虑到顺序表是否为空，如果是空表了，那么就无法删除了。
\begin{lstlisting}[language=C++]
bool delete_list(sq_list &L, int i, ElementType &e) {
  if (i < 1 || i > L.length) {
    return false; // 删除位置不合法
  }
  if (L.length == 0) {
    return false; // 顺序表为空，无法删除
  }
  e = L.data[i - 1]; // 获取被删除元素的值
  for (int j = i; j < L.length; j++) {
    L.data[j - 1] = L.data[j]; // 将删除位置之后的元素向前移动一位
  }
  L.length--; // 更新顺序表长度
  return true; // 删除成功
}
\end{lstlisting}
最好的删除情况是直接在表尾删除数据，这样不需要向前移动元素，时间复杂度为O(1)。 \\
最坏的删除情况是在表头删除数据，这样需要将所有元素都向前移动一位，时间复杂度为O(n)。平均情况下，删除操作的时间复杂度为O(n)。 \\
因此平均时间复杂度就是假设一个元素从第0位开始删除的复杂度加上第1位删除的复杂度加上第2位删除的复杂度...加上第n-1位删除的复杂度，最后除以n。
最后得到删除算法的平均时间复杂度为O(n)。

\subsubsection{获取元素操作}

这个就非常简单了，直接从头开始数，找到之后就返回位序，如果没有找到就返回false。 \\
如果在C++上可以用\code{std::find}算法来实现这个功能，或者直接使用\code{std::vector}的\code{std::find}算法来实现这个功能。 \\
自己进行查找，可以使用\code{std::optional<ElementType>}来表示查找结果，如果找到就返回元素值，如果没有找到就返回\code{std::nullopt}。
\begin{lstlisting}[language=C++]
std::optional<ElementType> get_data(sq_list &L, int i) {
  if (i < 1 || i > L.length) {
    return std::nullopt; // 获取位置不合法
  }
  return L.data[i - 1]; // 返回第i个元素的值
}
\end{lstlisting}
获取元素操作的时间复杂度为O(1)，因为顺序表支持随机访问，可以直接通过下标访问元素。

\subsubsection{按值查找}
按值查找需要从头开始遍历顺序表，直到找到第一个值等于e的元素为止，如果找到了就返回该元素的位序，如果没有找到就返回0。
\begin{lstlisting}[language=C++]
int locate(sq_list &L, ElementType e) {
  for (int i = 0; i < L.length; i++) {
    if (L.data[i] == e) {
      return i + 1; // 返回第i+1个元素的位序
    }
  }
  return 0; // 没有找到
}
\end{lstlisting}
按值查找的时间复杂度为O(n)，因为在最坏的情况下需要遍历整个顺序表才能找到目标元素或者确定目标元素不存在。

\begin{question}
\hd{2. 下列关于顺序表的叙述中，正确的是（ ）}
\begin{itemize}
  \item [A.] 顺序表可以利用一维数组表示，因此顺序表与一维数组在逻辑结构上是相同的。
  \item [B.] 在顺序表中，逻辑上相邻的元素物理位置上不一定相邻。
  \item [C.] 顺序表和一维数组一样，都可以进行随机存储。
  \item [D.] 在顺序表中，每个元素的类型不必相同。
\end{itemize}

\vspace{0.5em}

\begin{solution}
  顺序表是顺序存储的线性表，表中所有元素的类型必须相同，且必须连续存放。
  一维数组中的元素可以不连续存放。此外，栈、队列和树等逻辑结构也可以利用一维数组表示，
  但它与顺序表不属于相同的逻辑结构。在顺序表中，逻辑上相邻的元素物理位置也相邻。

  选项A的错误在于顺序表和一维数组在逻辑结构上是不同的，顺序表是一种线性结构，而一维数组是一种存储结构。 \\
  选项B的错误在于顺序表中逻辑上相邻的元素物理位置上也相邻。 \\
  选项C是正确的，顺序表和一维数组一样，都可以进行随机存储。 \\
  选项D的错误在于顺序表中每个元素的类型必须相同。
\end{solution}
\end{question}

\begin{question}
\hd {8. 在n个元素的线性表的数组中，时间复杂度为O(1)的操作是（ ）}
\begin{itemize}
  \item [I.] 访问第i（$1 \le i \le n$）个结点和求第i（$2 \le i \le n$）个结点的直接前驱
  \item [II.] 在最后一个结点后插入一个新元素
  \item [III.] 删除第一个结点
  \item [IV.] 在第i（$1 \le i \le n$）个结点后插入一个结点
\end{itemize}
\vspace{0.3em}
\medskip
\begin{tabular}{*{4}{p{0.22\linewidth}}}

\begin{question}
01. 从顺序表中删除具有最小值的元素（假设唯一）并由函数返回被删元素的值。
空出的位置由最后一个元素填补，若顺序表为空则显示出错信息并退出运行。

\vspace{0.5em}

\begin{solution}
  题目需要考虑的内容有三个，首先是如何找到最小值，并将其返回。
  其次是需要获取最后一个元素，可以手动计算也可以使用length获取，
  第三是需要确定顺序表是否为空，如果是空表了，那么就无法删除了。

  \begin{lstlisting}[language=C]
    //王道的参考答案代码:
    bool DEL_Min(SqList &L, ElemType &value){
      //删除循序表L中最小值元素结点，并通过引用型参数value返回其值
      //若删除成功，则返回true；否则返回false
      if(L.length == 0) return false //顺序表为空，无法删除
      value = L.data[0]; //假设第一个元素是最小值
      int pos = 0;
      for(int i=1; i<L.length; i++){
        if(L.data[i] < value){
          value = L.data[i]; //更新最小值
          pos = i; //记录最小值位置
        }
      }
      // 用最后一个元素填补空缺
      L.data[pos] = L.data[L.length - 1];
      L.length--;
      return true;
    }
  \end{lstlisting}

  \begin{lstlisting}[language=C++]
    //我自己写的代码，使用了C++的特性，代码更简洁一些。
    //同时因为使用std::optional，可以返回一个空值，
    //如果返回的是nullopt，那么就表示删除失败。
    template <typename T>
    std::optional<T> del_min(std::vector<T> &vec){
      if(vec.empty()) return std::nullopt;
      T min_value = vec[0];
      size_t pos = 0;
      for(size_t i = 1; i < vec.size(); ++i){
        if(vec[i] < min_value){
          min_value = vec[i];
          pos = i;
        }
      }
      // 用最后一个元素填补空缺
      vec.at(pos) = vec.back();
      vec.pop_back();
      return min_value;
    }
  \end{lstlisting}
\end{solution}
\end{question}

\begin{question}
  设计一个高校算法，将顺序表L的所有元素逆置，要求算法的空间复杂度为O(1)。

\begin{solution}
  算法思想：扫描顺序表L的前半部分元素，并将其与对应的后半部分元素交换位置，
  直到扫描到顺序表的中间位置为止。

  对于元素\code{L.data[i]}，其中0<=i<length/2，将其与后半部分的对应元素
  L.data[L.length - 1 - i]交换位置。

  \begin{lstlisting}[language=C++]
    //C语言版参考答案
    void Reverse(SqList &L){
      ElemType temp; //辅助变量
      for(int i=0; i<L.length/2; i++){
        temp = L.data[i]; //保存前半部分元素
        L.data[i] = L.data[L.length - 1 - i]; //将后半部分元素赋值给前半部分
        L.data[L.length - 1 - i] = temp; //将保存的前半部分元素赋值给后半部分
      }
    }
  \end{lstlisting}

  \begin{lstlisting}[language=C++]
    //C++版参考答案
    template <typename T>
    void reverse(std::vector<T> &vec){
      for(size_t i = 0; i < vec.size() / 2; ++i){
        std::swap(vec.at(i), vec.at(vec.size() - 1 - i));
      }
    }
  \end{lstlisting}
\end{solution}
\end{question}
